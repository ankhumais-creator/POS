üöÄ Installation Guide

Prerequisites

  • Node.js 18+ installed
  • Supabase account (free tier works)
  • Code editor (VS Code recommended)

Step 1: Extract & Install

# Extract the ZIP file to your desired location
# Navigate to the project directory
cd pos-kasir

# Install dependencies
npm install

Step 2: Supabase Setup

Info: Create a free Supabase account at supabase.com
  1. Create a new Supabase project
  2. Go to SQL Editor and run the schema file:
# Copy and execute the SQL from:
supabase/migrations/20260104000000_initial_schema.sql

Step 3: Environment Configuration

# Copy the example environment file
cp .env.example .env.local

# Edit .env.local with your Supabase credentials:
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
Warning: Never commit .env.local to version control! It's already in .gitignore

Step 4: Run the Application

# Development mode
npm run dev

# Production build
npm run build

# Preview production build
npm run preview
Success! Application should now be running at http://localhost:5173

üîå API Reference

Database Schema

The application uses the following main tables:

  • products - Product inventory
  • categories - Product categories
  • transactions - Sales transactions
  • transaction_items - Transaction line items
  • customers - Customer profiles
  • discounts - Discount/promo codes
  • shifts - Cashier shift records
  • stock_adjustments - Stock movement history

Offline Sync

The application uses a sync queue system:

  • All changes are saved locally in IndexedDB first
  • Changes are queued for sync when online
  • Automatic retry on connection restoration
  • Conflict resolution uses "last write wins" strategy

Troubleshooting

Build Errors

# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

Supabase Connection Issues

  • Verify environment variables are correct
  • Check Supabase project is active
  • Ensure API keys have proper permissions
  • Verify SQL schema was executed completely

Data Not Syncing

  • Check browser console for errors
  • Verify internet connection
  • Check Supabase service status
  • Review sync queue in IndexedDB
